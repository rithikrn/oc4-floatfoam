# Irregular PM-wave case

This case applies a Pierson–Moskowitz irregular-wave realization. The `constant/waveProperties` dictionary is generated by `tools/generate-pm-waveproperties.py`.

---

## What this case does

- Runs `interFoam` with dynamic mesh motion and `sixDoFRigidBodyMotion`.
- Uses `irregularMultiDirectional` wave forcing at the inlet.
- Uses `shallowWaterAbsorption` at the outlet.
- Includes `prepare-waves.sh` to regenerate `constant/waveProperties`.
- Writes forces, six-DOF state, interface-height probes, vorticity, and Q-criterion outputs.

---

## Default PM setup

The shipped `constant/waveProperties` corresponds to:

```text
Hs      = 2.44 m
Ts      = 8.1 s, treated as Tp
N       = 100 components
seed    = 12345
f_lo    = 0.5 fp
f_hi    = 4.0 fp
ramp    = 4 Ts = 32.4 s
```

## Regenerate waveProperties

From this case directory:

```bash
./prepare-waves.sh
```

Change the sea state:

```bash
HS=5.49 TS=11.3 NCOMP=100 SEED=42 F_LO=0.5 F_HI=4.0 RAMP=4 ./prepare-waves.sh
```

Create the optional verification plot:

```bash
PLOT=1 ./prepare-waves.sh
```

Outputs:

```text
constant/waveProperties
constant/PM_spectrum_verification.png   # only when PLOT=1
```

---

## Motion constraints

This case uses a constrained `sixDoFRigidBodyMotion` setup, not a fully free six-DOF/moored model:

- `fixedLine` direction `(0 0 1)` allows heave.
- `fixedAxis` axis `(0 1 0)` allows pitch.
- sway, surge, roll, and yaw are locked.
- the active restraint is a vertical `linearSpring`; no explicit mooring-line model is enabled.

The generated `0/` directory is not tracked in Git. Always run `./mesh.sh` first on a fresh clone; it copies `0.orig/` to `0/`, applies `setFields`, and decomposes the initialized fields.

## Run

```bash
cd cases/irregular
module load openfoam

./prepare-waves.sh      # optional if using the shipped waveProperties
./mesh.sh
./run.sh
reconstructPar
```

Slurm:

```bash
sbatch submit.slurm
```

Regenerate PM waves inside the Slurm job:

```bash
PREPARE_WAVES=1 sbatch submit.slurm
```

---

## Main files to inspect

- `constant/waveProperties`: generated PM irregular-wave components.
- `prepare-waves.sh`: wrapper around the Python generator.
- `0.orig/U`: wave velocity boundary conditions.
- `0.orig/alpha.water`: wave phase-fraction boundary conditions.
- `system/controlDict`: forces, six-DOF state, probes, vorticity, Q-criterion.

---

## When to regenerate

Regenerate `waveProperties` when changing:

- significant wave height `HS`;
- peak period `TS`;
- number of components `NCOMP`;
- random seed `SEED`;
- frequency range `F_LO` / `F_HI`;
- ramp multiplier `RAMP`.

After changing waves, clean old run artifacts before rerunning:

```bash
rm -rf 0 processor* postProcessing log.*
./mesh.sh
./run.sh
./reconstruct.sh
```

More details are in [`../../tools/README.md`](../../tools/README.md).
